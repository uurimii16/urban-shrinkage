## 1. 오늘 한 일 요약

기존 `app.py`(검증 완료 원본)는 **그대로 두고**, UX를 새로 짠 **`app_v2.py`** 를 만들어 발전시켰습니다.
검증된 엔진 모듈(decline_engine/legal_engine/loader/golden_io/custom_indicators)은 **손대지 않음** → 골든 검증 유지.

### (1) UX 리디자인 — `app_v2.py`
- **단계형 스테퍼**: ① 데이터 입력 → ② 설정 → ③ 시트 검토 → ④ 산출. 한 번에 한 단계, 상단 진행바.
- 설정(가중치·등급)을 **좁은 사이드바 → 본문 넓은 패널**로 이동. 사이드바는 진행 상태 요약.
- 결과 화면에 **요약 타일 + 행정동 순위 차트(altair) + 탭 표**.
- 단계 이동에도 데이터/결과 유지(session_state).

### (2) 차장님 피드백 반영 (A~D)
- **A. 2024 경고 버그 수정**: 사용자가 고른 **기준연도를 경고 계산 직전에 반영**. (기존엔 항상 2024 기준으로 잘못 떴음)
- **B. 소형주택 기본 제외 + 지표 사용 on/off**: 각 지표 '사용' 체크박스. 끄면 종합/엑셀에서 제외.
- **C. 가중치 손쉬운 조정**: '⚖ 부문 합 100 자동 맞춤' 버튼(비율 유지 정규화) + 부문별 합 실시간 표시.
- **D. 복합·피벗 집계구/행정동 선택**: ③단계 '복합·피벗 집계 단위'(행정동/집계구/둘다). 수식모드 SUMIFS 참조열 자동 분기(행정동=$G, 집계구=$B).

### (3) 계산식 기반 사용자 지표 엔진 (E) — `recipe_engine.py` (신규)
- 지표를 '함수(코드박음)'가 아니라 **'레시피(표 한 줄)'**로 정의 → **UI에서 자유 추가/삭제/수정**.
- ②설정 → **🧮 지표 계산식 관리**: 기본 12지표 수식 읽기전용 표시 + 사용자 지표 추가(＋행)/삭제(🗑).
- 레시피: 유형(비율/합계/증감률) · 카테고리(in_age 등) · 분자/분모 코드범위 · 스케일 · 방향 · 부문.
- 예: `20~39세여성비율 = Σ(in_age 65~68) ÷ Σ(1~21) × 100` → 수동계산과 **오차 0.0** 검증.
- 만든 지표는 가중치 표에 합류 → 종합점수·엑셀까지 자동 반영.

### (4) 최종 4개 시트만 출력 옵션 (오늘 추가)
- ④산출에 체크박스 **"최종 4개 시트만 출력"**. 중간 DATA/피벗 생략, **자기완결형 값**으로 4개만:
  법적쇠퇴진단(행정동/집계구) + 복합쇠퇴지수(행정동/집계구) — **유림_17시 3~6번 대응**.
- 파일 14MB → 0.5MB로 가벼움. 외부참조/수식 0 확인.

### (5) 검증
- 골든본(=유림_17시)과 셀단위 대조: **복합쇠퇴지수(행정동) 1360/1360 = 100% 일치** (`verify_jeonju.py`).
- ⚠️ 값·Z·T는 가중치와 무관 → 유림과 정확히 일치. **종합점수·등급은 가중치가 다르면 당연히 다름**(앱 기본 40/20/40 vs 유림 남원값). 값 대조는 **값·Z·T 위주**로.

## 3. 다음에 할 일 (집에서)

### 결정된 것
- **복제 방식 확정**: 기본 12지표는 검증 그대로 두고, 화면엔 수식 표시 + **'복제해서 수정'**(사본 만들어 고치고 원본 끄기). 원본 보존·되돌리기 가능.

### 다음 작업 = 지표 관리 UI **통합** (아직 미구현)
지금 지표가 세 갈래(①기본 ②계산식 ③추가지표 파일)로 흩어져 혼란 → **한 표로 통합**:
- 마스터 표 하나에 모든 지표(기본/계산식/값) = 부문·방향·**사용·내부가중치**를 한 곳에서.
- 표에서 **행 추가/삭제**(사용자 지표), 기본지표는 **복제**로 편집.
- **추가지표(파일)** = "이미 계산된 값을 가져오는 버튼"으로 흡수(별도 개념 아님).
  - 형식: `지표명|부문|방향|단위레벨|단위코드|값|내부가중치` (한 줄=한 지표×한 지역). 단위코드는 앱 지역코드와 일치해야 함.

### 남은 검토거리
- **경고/차단 완화**: 6분류 다 있어야 최종 산출되는 하드차단 → "**사용 중 지표가 필요한 데이터만**" 검사로. in_age 없으면 관련 지표 자동 제외하고 진행(막지 말고 확인만).
- **코드표 안전장치**: 현재 기본지표는 코드 004~013을 **의미 모르고 그대로 연산**(positional). 업로드 코드표는 표시용일 뿐 계산 무관. → (가) 코드범위 검증 경고부터, 여력되면 (나) 의미기반 매핑.
- **미정 질문**: 계산식 입력방식(구조화 빌더=현재 vs 텍스트 수식), 기본지표 직접편집 허용 여부 → 복제방식으로 정리됨.
