## 3. 두 진단 로직 (골든본에서 역설계·검증 완료)

### 법적쇠퇴진단 (`legal_engine.py`) — 규칙 기반 이진 판정 (표준화·가중치 없음)
- **인구**: 30년 최다 대비 증감률 ≤ −20% → o, 또는 최근5년 3년연속감소 → o
- **산업**: 총사업체수(to_fa_010) 증감률 < −5% → o, 또는 3년연속감소 → o
- **물리**: 노후건축물(2004이전 = ho_yr_001~004)/전체 ≥ 50% → o
- **부합개수** = 3부문 각 1점(부문 내 OR), **쇠퇴지역 = 부합 ≥ 2** (도시재생법 시행령 방식)
- 입력: to_in, to_fa, ho_yr

### 복합쇠퇴지수 (`decline_engine.py`) — 연속 가중 표준화 점수
- 12지표: 인문사회(인구변화율·노년부양비·경제활동인구비율·소멸위험지수) /
  산업경제(총사업체·총종사자·제조·고차·도소매·음식숙박 증감률) / 물리환경(노후건축물비율·소형주택비율)
- Z=(값−평균)/모표준편차(ddof=0), T=Z×방향부호(±10)+50. **+10은 노년부양비·노후건축물·소형주택 3개만**.
- 부문점수=Σ(T×가중치), 종합=Σ(부문), 10등급(Jenks 기본/Quantile/Pretty).
- 산업 8/9/10차 분기: 제조 8차 004→9·10차 003, 음식숙박 008→009, 고차 9·10차 +017, 도소매 007.
- 노후=ho_yr 001~004(2004이전), 소형=ho_ar 001~003(60㎡이하). 총인구 분모=Σ(in_age 001~021).

## 4. 골든본 20시트 구조 & 의존관계 (#7 빌더용 청사진)

```
원시 항목코드(long)
 ├ to_in_001 ─▶ 법적DATA_1.인구총괄(수정)      ─┐
 ├ to_fa_010 ─▶ 법적DATA_2.총괄사업체수(수정)   ─┼─▶ 전주시 법적쇠퇴진단(집계구) ─SUMIFS▶ (행정동)  [최종]
 ├ ho_yr_*  ─▶ 법적DATA_3.건축연도별주택(수정) ─┘        (VLOOKUP 연도&집계구, 임계값 o/x)
 ├ in_age_* ─▶ 복합DATA_1.인문사회(수정) ─▶(행정동)피벗 BQ노년/BR경활/BS소멸 ─┐
 ├ cp_bem_* ─▶ 복합DATA_2.종사자수(수정) ─▶(행정동)5블록 증감률 ───────────┤
 ├ to_fa_010─▶ 복합DATA_2.총괄사업체수(수정)─▶(행정동)증감률 ─────────────┼─▶ 전주시 복합쇠퇴지수(집계구/행정동)
 ├ ho_ar_*  ─▶ 복합DATA_3.소형주택비율(수정)─▶(행정동)소형비율 ───────────┤   (Z/T·가중치·종합·10등급,
 └ ho_yr_*  ─▶ 복합DATA_3.주택건축물비율(수정)▶(행정동)노후비율 ──────────┘    D열 인구변화율=법적진단!R 재사용)
lookup: 행정동수정_금암동_5186 (행정동코드→행정동명, VLOOKUP)
```
- **원시-파생 입력 시트(long+임베디드 피벗)**: 법적DATA_1/2/3, 복합DATA_1/2사업체/2종사자/3소형/3건축.
- **중간 피벗(행정동 SUMIFS)**: 복합DATA_*(행정동) 5종.
- **최종 출력**: 법적쇠퇴진단(집계구/행정동), 복합쇠퇴지수(집계구/행정동).
- 시트 헤더가 **행 1~4가 아니라 특정 행**(법적진단은 헤더 행10~12·데이터 행13)인 점 주의.
- 열 위치 하드코딩 상수(재현 시 보존): 법적DATA_1 값=C:E의 3열, 법적DATA_2 값=F:G의 2열,
  법적DATA_3 노후=I:AH의 23열/전체=22열; 임계값 −20/−5/≥50, 부합≥2.
- **연도집계구 결합키** = 연도4자리 + 집계구14자리 문자열(예 `202435011600010001`).
