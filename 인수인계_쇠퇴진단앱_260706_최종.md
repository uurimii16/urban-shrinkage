
## 1. 앱 흐름 = 5단계 (엑셀 탭처럼 클릭 이동)
```
① 자료 신청 → ② 데이터 입력 → ③ 설정(가중치) → ④ 시트 검토 → ⑤ 산출
```
- **① 자료 신청**: SGIS 집계구 자료를 신청(제출)까지 자동. (쿠키 붙여넣기 + 지역·항목 체크 → 신청)
- **② 데이터 입력**: 받은 zip/폴더/파일을 넣음(zip 압축 그대로 OK, 드래그앤드롭 지원). 집계구14→행정동8 자동 롤업.
- **③ 설정**: 지표 마스터(사용·가중치)·계산식·값파일·정렬 등.
- **④ 시트 검토 / ⑤ 산출**: 통합 Excel(법적·복합 진단) 생성·다운로드.

## 2. 오늘(07-06) 추가·변경 요약
- **UI 5단계 재편** + 탭 클릭 이동(①자료신청 신설, 나머지 한 칸씩 밀림).
- **SGIS 자료신청 자동화**(`sgis_request.py` + ①단계): 쿠키(로그인 출입증) 붙여넣기 + 지역·항목 체크 → `saveRequestData`로 **장바구니 통째 신청**. 항목 하나하나 담는 노가다 제거. **전 연도 명시 신청**(인구·주택 13개년: 2000·05·10·15+2016~2024매년 / 사업체 25개년 매년, 차수코드 연도별 자동). 다운로드는 승인(약10분) 이메일 후 수동.
- **집계구 API 불가 확정**: SGIS OpenAPI는 행정구역까지만 → 집계구는 자료제공 신청만. (API 수집기 `sgis_collect.py`는 행정동 시계열 보조용.)
- **zip 그대로 입력**: 업로드/폴더경로/드래그앤드롭 모두 zip 자동해제(`loader._expand_uploads`/`_zip_members`).
- **행정동명 붙이기**: 집계구[:8]=행정동코드 자동. 이름은 전주 34동 기본 내장(`dong_names.py`) + 매핑 CSV(어느 지역이든, 2열/3열 자동인식 `parse_dong_mapping`). **집계구 시트에도 행정동명 표시**(export/sheet_builder [:8] 폴백, 집계구시트 name_map 전달).
- **⑤ 산출 정렬 옵션**: 행정동코드 오름/내림, 종합점수 높은/낮은순(엑셀·표 행순서).
- **연도 자동 오름차순**: 멀티셀렉트에서 지웠다 넣어도 항상 연도순 처리("사용 연도(정렬됨)" 표시).
- **계산식 샘플 3종 미리 로드**(`recipe_engine.sample_recipes` — 비율/증감률/합계, 기본 사용해제) + 편집기 **추가/삭제 버튼**(➕ 한 줄 추가 / 🗑 체크 삭제 / 🎓 샘플 재삽입).
- **내부가중치 균등분배 버튼**(`equalize_weights`): 부문 사용지표에 100 균등(5개면 각 20). 기존 '비율유지 100'과 별도.
- **개인정보 코드에서 제거**(공개배포 대비): 신청자 정보는 앱 ①화면 입력 → `submit_cart(applicant=)`.
- **성능 최적화**: 업로드 파일 로딩 bytes캐시(9번). **안 잠들기** GitHub Actions(8번). **GitHub 배포**: `uurimii16/urban-shrinkage`.

## 3. SGIS 데이터 받는 법(집계구, 한 번만)
- ① 자료신청 화면: SGIS(sgis.mods.go.kr) 로그인 → F12 → Network → 새로고침 → 아무 요청 Copy as cURL → 쿠키칸에 붙여넣기 → 지역(시군구코드)·항목 확인 → **신청**.
- 필수 5종(=복합쇠퇴 전체세트): 총인구·성연령별인구·건축년도별주택·종사자수(전연도)·총괄사업체수(전연도). (총괄종사자수는 불필요 — 대분류 합산으로 계산)
- 약 10분 뒤 **SGIS 승인 이메일** → 다운로드(zip) → ② 데이터 입력에 그대로 투입.

## 8. 안 잠들기(Keep Awake) — 무료 Streamlit 자동 sleep 방지
- 무료 배포앱은 미사용 시 잠듦 → GitHub Actions가 **6시간마다 방문·깨움**.
- 파일: `.github/keep_awake.py`(playwright로 방문, "get this app back up" 버튼 자동 클릭) + `.github/workflows/keep-awake.yml`(cron `0 */6 * * *`).
- **배포 후 할 일**: repo Settings → Secrets and variables → Actions → **Variables** 탭 → `STREAMLIT_URL` 을 실제 배포 주소로 등록(없으면 yml 기본값 `https://urban-shrinkage.streamlit.app/` 사용). Actions 탭에서 수동 실행(workflow_dispatch)으로 한 번 테스트 가능.

## 9. 성능 최적화(반응 속도)
- 업로드 원시파일 로딩을 **bytes 기준 캐시**(`cached_load_uploads` + `_files_sig`) → 파일 넣고 설정 만질 때마다 재파싱하던 것 제거(체감 큼).
- 폴더/워크북 로딩은 기존대로 `@st.cache_data`. 엔진 계산(E.run 등)은 config 전역(기준연도)에 의존해 캐시하지 않음(정확성 우선) — 산출 버튼 눌렀을 때만 1회 실행.
