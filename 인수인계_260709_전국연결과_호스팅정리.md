## 1. 행정구역 전국 코드표 → dong_names 연결 ✅ (완료·커밋됨)

### 무엇을
`행정구역코드/행정구역코드_전국.xlsx`(SGIS 전국 3,559 행정동, 이미 저장소에 포함·추적됨)를
`dong_names.py`가 **자동으로 읽어**, `default_name_map()`이 전주 34동 대신 **전국 행정동 이름**을
반환하도록 확장. 이걸 쓰는 모든 진입점(`batch_run`·`batch_watch`·`app_v2`)이 **코드 수정 없이**
전국 대응이 됨(결과표에 코드 대신 "노송동·가덕도동" 등 이름 표시).

### 바뀐 파일
- `dong_names.py` (+89/-7): 전국 xlsx 로드 + 캐시 + 폴백. 신규 함수:
  - `default_name_map()` — 전국표 있으면 3,559동, 없거나 openpyxl 부재면 **전주 34동 폴백**(하위호환).
  - `default_sigungu_map()`(시군구5→명, 252개), `default_sido_map()`(시도2→명, 17개), `national_loaded()`.
  - 캐시(`_CACHE`)로 배치에서 시군구마다 불러도 **xlsx는 1회만 읽음**.
- `batch_run.py` (+1/-1): `sido_name_map=None` → `dong_names.default_sido_map() or None` 으로 채움.
- `행정구역코드/README.md`: "5-1 파이프라인 연결(완료)" 기록, 법정동 후보는 사용자 요청으로 제외.

### 검증(통과)
- 전국 로드: 행정동 3,559 · 시군구 252 · 시도 17
- 실코드 조회 정상: 서울(개포1동)·부산(가덕도동)·대구(군위읍)·인천(강화읍)·광주(도산동)·전주(노송동)
- 폴백 하위호환: xlsx 없으면 34동, `national_loaded()=False` 확인
- `py_compile` OK

### 주의
- 읍면동 개편 시 `행정구역코드/sgis_admin_codes.py` 재실행으로 xlsx 갱신하면 이름표도 자동 최신화(앱 재시작 시 반영).
- **배포앱도 전국 이름 나옴**: 전국 xlsx가 저장소에 추적돼 있어 Cloudtype 배포본에도 포함됨.

## 2. Cloudtype 호스팅 메모리 충돌 해결 🔶 (거의 완료, TODO 1개 남음)

### 증상
쇠퇴진단 앱이 안 뜸. 에러: `Memory limit exceeded ... 0.5GB requested, but 0GB remained`.
→ keep-awake 문제 아님. **컨테이너가 메모리 자리가 없어 시작조차 못 한 것**(앱 코드 실행 전 단계).

### 원인
- Cloudtype **무료 스페이스**는 상시 가동 메모리 총량이 작음(≈0.5GB 경쟁).
- 같은 스페이스에 국내 IP 필수 앱 **2개**가 물려 있었음:
  - `urban-shrinkage`(쇠퇴진단, SGIS 해외IP차단 때문에 국내 호스팅 필요)
  - `gis-address-tool`(주소변환기, VWorld 해외IP차단 때문에 국내 호스팅 필요)
- 게다가 쇠퇴진단이 **워크로드 중복**(`urban-shrinkage` + `app` 두 개)으로 떠 있어 메모리를 이중 점유.

### Oracle 이전 시도 → 중단
gis를 Oracle Cloud 평생무료(한국 리전)로 분리하려 했으나, **무료 계정 홈 리전에 한국이 안 뜨고
일본만 제공됨**. 일본=해외 IP라 VWorld가 막혀 무의미 → Oracle 경로 폐기.
(잔재: `gis-repo/deploy/`, `gis-repo/ORACLE_이전_가이드.md` — **미사용, 지워도 됨**.)

### 최종 해결: Cloudtype 하비 구독
- **하비(Hobby) 6,600원/월 = 512MB "상시슬롯 1개"**(하루1회 강제중지·절전 없음).
- ⚠️ **구독만으론 자동 적용 안 됨** — 서비스마다 메모리를 프리→구독으로 직접 바꿔야 함.
- **`urban-shrinkage` 워크로드**(= 쇠퇴진단 메인 사이트, 소스 `github.com/uurimii16/urban-shrinkage.git`,
  실행 `streamlit run app_v2.py`)를 **구독 리소스로 전환 → 상시가동**.
- 중복 **`app` 워크로드는 중지**(삭제 아님, 되돌리기 쉬움).
- `gis-address-tool`은 **프리 유지**(하루1회 중지, 쓸 때 깨움). 늘 켜두려면 512MB 하나 더(합 13,200원).

### 워크로드 최종 상태(목표)
| 워크로드 | 정체 | 메모리 | 비고 |
|---|---|---|---|
| `urban-shrinkage` | 쇠퇴진단 메인 사이트 | **구독(하비)** | 동료 공유 URL은 이 워크로드 것 |
| `gis-address-tool` | 주소변환기 | 프리 | 온디맨드 |
| `app` | 쇠퇴진단 **중복** | (중지됨) | TODO 후 삭제 가능 |

## 5. 기억해둘 사실
- **구독은 워크로드에 걸림** — 중복 삭제해도 구독 안 날아감. 오히려 중복 지워야 메모리 풀림.
- 512MB로 분석 중 앱이 죽으면 **1GB(13,200)로 클릭 승급**. 무거운 전국 배치는 **로컬 `batch_run.py`**.
- 워크로드 이름 정리: **`urban-shrinkage`(dockerfile, app_v2.py) = 쇠퇴진단 사이트**. "urban-shrinkage"는 프로젝트명이자 그 워크로드명.
- 전국출력 배치 방식: 시군구마다 xlsx 1개씩 → zip 하나 + `_요약.csv`(시군구별 독립 표준화, 광역시는 시 통째).
