## 2. 복합종합 수식 규칙 (N=마지막행=2+집계구수)
- 지표 i(0~11): 값열=4+3i, Z열=5+3i, T열=6+3i. (i0 인구변화율 D/E/F … i11 소형주택 AK/AL/AM)
- Z(i>0)=`=IF(V="",0,((V-AVERAGE(V$3:V$N))/_xlfn.STDEV.P(V$3:V$N)))`, i0=IF없음.
- T=`=(Z*방향부호)+50` (i>0은 IF감쌈). 방향: 노년부양비·노후건축·소형주택=+10, 나머지=-10.
- 인문사회 AN=`=SUM(F*계산방법!$E$24, I*$E$25, L*$E$26, O*$E$27)`, 에러수정 AO=IFERROR판. 산업경제=R/U/X/AA/AD/AG×E28~33. 물리=AJ/AM×E34/35. **종합 AT=`=SUM(AO,AQ,AS)`**. (계산방법 E행=24+지표순번)
- 등급 전부 **종합(AT)열 기준**: 등분위·등간격=함수, Natural Break=엑셀함수無→파이썬 Jenks경계 계산 후 경계+VLOOKUP(추천) 또는 값.

## 4-1. 추가 완료 (2026-07-09) — ③설정 가중치 → 정본 양식 연결
- **문제였던 것**: `template_export`가 하드코딩 상수(`TEMPLATE_INDICATORS`)만 써서 앱 ③설정 가중치와 무관하게 산출됐음.
- **연결 방법**: `template_export.indicators_from_cfg(cfg)` 신규 — 앱 `ss.cfg`(indicator_ids·sign_map·final_weight·sector_of·label_map) → `(id,헤더,부문,방향,가중치)` 리스트로 변환. 헤더는 정본 헤더 우선, 커스텀은 label_map.
- `compute_values(raw_sub, ids, key)`·`build_composite_workbook(raw_sub, admin_path=None, indicators)` 를 지표목록 기반으로 일반화(값·계산방법·종합 3곳 동일 목록 → E열 참조 자동 일치). admin_path 기본값=`DEFAULT_ADMIN_PATH`(=행정구역코드/행정구역코드_전국.xlsx).
- **앱 연결**: `app_v2.py` STEP 산출 결과화면에 **"정본 양식 생성(가중치 반영)" 버튼 + 다운로드** 추가(`import template_export as TE`). 클릭 시 `TE.indicators_from_cfg(ss.cfg)` → `TE.build_composite_workbook(raw_selected, ...)`.
- **커스텀/계산식 지표 값 채움 해결(재계산 없이 재활용)**: 세 엔진 모두 `(지표,'값')` 원값 컬럼 생성(decline `build_scores`·recipe `build_recipe_scores`·custom `build_scores`) → 앱 산출 때 `jgu_scores = combine_scores(combine_scores(jgu_base[0], custom_jgu), recipe_jgu)` 에 base+custom+recipe 전체 집계구 원값이 이미 있음.
  - `template_export.values_from_scores(scores, ids)` 신규: scores에서 `(지표,'값')`만 뽑아 정본 지표값 DF. `build_composite_workbook(..., values=...)` 인자 추가(주면 재계산 안 함, 없으면 raw에서 기본지표만).
  - `app_v2.py`: 산출 블록에서 `ss.results['template_values']=TE.values_from_scores(jgu_scores, indicator_ids)` 저장 → 정본 버튼이 `values=res.get('template_values')`로 사용. **커스텀/계산식 지표값도 채워짐(검증: 상권활력지수 72.4 등 D/G열 기입 확인).**
- **검증**: 합성 cfg(3지표·가중치 0.5/0.2/0.3·방향 -10/-10/+10)로 계산방법 E24~E26 = 설정값, 종합 부문셀이 각각 $E$24/25/26 참조 확인. 별도로 base+custom 혼합 scores→값 채움 확인.
- **주의/남은 것**: 아직 **계산방법+복합종합 2시트만**(정본 9시트 아님). 지표를 끄면 열 수가 줄어 정본 49열과 달라짐(내부 함수는 일치). 앱 기본은 11지표(소형주택 제외)이므로 정본 12지표를 원하면 ③설정에서 소형주택 켜기. 법적종합·원시6시트·NaturalBreak(4번 항목)는 여전히 미완.

## 4-2. 추가 완료 (2026-07-09) — "선택 시군구 개별 산출" (구 '전국 배치 빌드' 개조)
- **문제**: ② 데이터입력의 '🌏 전국 시군구 배치 빌드' 는 위치가 ③설정 앞이라 **디폴트 가중치·구양식**만 가능(어중간·전국원스톱과 겹침). 이름도 애매.
- **결정(B안)**: 삭제 대신 **⑤ 산출로 이동 + 복수선택 + ③설정 가중치 + 정본 양식 + 따로 저장**으로 개조. 이름 **"선택 시군구 개별 산출 (내 설정 적용)"**.
- **batch_build.py**: `build_one_template(raw_sub, indicators, custom_df, recipes, admin_path)` 신규 — 시군구 집계구 base scores(E.run jgu) + 커스텀(CI.build_scores) + 계산식(RE.build_recipe_scores) 합쳐 `TE.values_from_scores` → `TE.build_composite_workbook`. `build_batch_zip(..., template_mode=True, indicators, custom_df, recipes, admin_path)` 분기 추가(기존 구양식 경로·전국원스톱·batch_run은 무영향).
- **app_v2.py**: ② 데이터입력의 옛 expander 제거(⑤로 안내 캡션만). ⑤ `step4_run`에 `🏙 선택 시군구 개별 산출` expander 추가 — `st.multiselect`로 시군구 고름 → `BB.build_batch_zip(raw_selected, sigungu=pick, template_mode=True, indicators=TE.indicators_from_cfg(ss.cfg), custom_df=ss.custom_df, recipes=active_recipes, ...)` → zip/폴더 개별 저장.
- **indicators_from_cfg 폴백 보강**: cfg에 sign/weight/sector 없으면 정본 기본값(_TEMPLATE_SIGN/_TEMPLATE_W/_TEMPLATE_SEC)으로. (빈 cfg여도 기본 12지표 가중치 정상)
- **검증(실데이터 전주)**: golden_io로 로드 → 35011(695집계구)·35012(671집계구) 각각 정본 2시트 OK, 행정동명 자동조회(동서학동/인후1동), cfg 가중치 E24=0.7 반영, 빈cfg=기본가중치 폴백 확인.
- **주의**: 정본 배치도 아직 **계산방법+복합종합 2시트**(9시트 아님). 커스텀지표가 행정동 단위면 집계구 배치와 안 맞을 수 있음(집계구 단위 값만 반영). 표준화는 시군구별 독립(정본 종합 Z/T가 그 파일 자기 행으로 함수계산).
