# 복합쇠퇴진단 자동화 앱

SGIS 원시 CSV 또는 정제 통계 xlsx를 넣으면 **법적 data / 복합 data 개별 시트**와
**법적쇠퇴진단 + 복합쇠퇴지수**를 통합 xlsx로 만드는 앱. 계산 엔진은 전주시 골든본
(`260415_전주시 복합쇠퇴진단 기초자료…xlsx`)의 행정동 값·Z·T·부문·종합을 **100% 재현** 검증됨.

## 구성

| 파일 | 역할 |
|---|---|
| `config.py` | 지자체 상수(기준연도·연령계급·주택코드·산업 8/9/10차 분기·12지표 가중치·부호) |
| `decline_engine.py` | 순수 계산 엔진. `run(raw, level='dong'|'jgu', grade_method, n_classes)` |
| `golden_io.py` | 골든본/정제 xlsx의 6개 `(수정)` 시트 → 원시 long dict 추출(부작용 없음) |
| `sheet_builder.py` | 원시 long dict → 법적/복합 DATA 개별 시트·행정동 피벗·최종표 통합 워크북 생성 |
| `custom_indicators.py` | 사용자 추가지표 템플릿·검증·표준화·복합점수 합산 |
| `export.py` | 엔진 결과/통합 워크북 xlsx 생성 진입점 |
| `legal_engine.py` | **법적쇠퇴진단**(도시재생법 방식, 규칙 기반 o/x). `run_legal(raw, level)` |
| `loader.py` | 원시 SGIS CSV 폴더/업로드 파일(cp949 등) → 법적 data long dict |
| `verify_jeonju.py` | 골든본 복합쇠퇴지수 셀단위 대조(Stage 1 앵커) |
| `verify_loader.py` | 원시 CSV→loader→엔진 결과를 골든과 대조(Stage 2, Part A) |
| `verify_sheet_builder.py` | 골든 파일 없이 통합 시트 빌더 최소 회귀 검증 |
| `verify_custom_indicators.py` | 추가지표 + 수식 엑셀 생성 경로 최소 회귀 검증 |
| `app.py` | Streamlit UI(카테고리별 CSV/폴더/xlsx → 개별 시트 검토 → 통합 xlsx 다운로드) |

## 실행

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 1) 앵커 검증(골든본 재현 100% 확인) — data/golden.xlsx 필요
PYTHONUTF8=1 .venv/bin/python verify_jeonju.py

# 2) 통합 시트 빌더 최소 검증 — 골든본 불필요
PYTHONUTF8=1 .venv/bin/python verify_sheet_builder.py
PYTHONUTF8=1 .venv/bin/python verify_custom_indicators.py

# 3) 앱 실행
.venv/bin/streamlit run app.py
```

`data/golden.xlsx` 는 골든본 원본을 복사한 것(gitignore). 다른 지역은 같은 시트 구조의
정제 xlsx를 앱에 업로드하면 됨.

## 두 가지 진단 (둘 다 골든본 100% 검증)

- **법적쇠퇴진단** (`legal_engine.py`): 표준화·가중치 없는 규칙 판정.
  인구 증감률 ≤ −20% 또는 3년연속감소 / 사업체 증감률 < −5% 또는 3년연속감소 /
  노후건축물비율 ≥ 50% → 각 부문 1점, **부합 ≥ 2 = 쇠퇴지역**. (골든 34/34 일치)
- **복합쇠퇴지수** (`decline_engine.py`): 12지표 Z/T·가중치·10등급. (골든 1360/1360 일치)

## 지표 · 계산 (요약)

- 12지표: 인문사회(인구변화율·노년부양비·경제활동인구비율·소멸위험지수) / 산업경제(총사업체·총종사자·제조·고차·도소매·음식숙박 증감률) / 물리환경(노후건축물비율·소형주택비율)
- 표준화 Z=(값-평균)/모표준편차(ddof=0), T=Z×방향부호(±10)+50. **+10은 노년부양비·노후건축물·소형주택 3개만**.
- 부문점수=Σ(T×가중치), 종합=Σ(부문). 등급 10단계 = Jenks(기본)/Quantile/Pretty.

## 상태

- **Phase 1 완료**: 엔진(집계구+행정동)·출력 xlsx·Streamlit UI. 골든본 Stage 1 앵커 검증
  **1360/1360 = 100%**.
- **Phase 2 완료(1개 지표 데이터 갭)**: 원시 CSV 로더 + 앱 원시폴더 입력 모드.
  `verify_loader.py` 기준 **12지표 중 11개가 원시 CSV에서 골든 100% 재현**.
  - 남은 1개(**총사업체수 증감률**): USB에 총괄사업체수 원본이 **2004~2015만** 존재하고
    2000~2003·2016~2023(기준연도 2023 포함) 파일이 없어 골든과 불일치. cp_bnu 합으로는
    복원 불가(소규모 셀 N/A로 과소집계). **로더 코드 문제가 아니라 원본 파일 부족**이며,
    해당 연도 총괄사업체수 CSV를 폴더에 추가하면 자동 해소됨.
  - 원시 경로 입력 시 `loader.coverage_warnings()`가 기준연도 부재를 앱에서 경고.
- **Phase 3 진행**: 법적/복합 DATA 개별 시트 빌더와 통합 UI 추가.
  - 앱에서 CSV 폴더 경로, 카테고리별 CSV 파일 업로드, 기존 법적 data xlsx 업로드를 선택 가능.
  - 출력/산출 연도를 선택하고, 법적 DATA·복합 DATA·행정동 피벗·최종 진단 시트 포함 여부를 선택 가능.
  - 파일 업로드 모드에서는 SGIS 기본 4열 외 구조도 컬럼 매핑 보정 가능.
  - 35011/35012처럼 시군구별로 쪼개진 CSV는 집계구코드 기준으로 자동 통합하고, 파일 수·행 수·연도 커버리지를 앱에서 표시.
  - 가중치는 부문비율(기본 40/20/40)과 부문 내부 지표비율 2단 구조로 조정.
  - 추가지표 xlsx/csv 업로드 지원. 형식은 `지표명, 부문, 방향, 단위레벨, 단위코드, 값, 내부가중치`.
  - 다운로드는 `수식 엑셀`과 `값 엑셀` 중 선택 가능. 수식 엑셀은 최종 복합표의 값/Z/T/가중합을 엑셀 수식으로 생성.
  - 추가지표 템플릿과 집계구-행정동 매핑 CSV 템플릿을 앱에서 다운로드 가능.
