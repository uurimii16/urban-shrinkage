# SGIS 자료제공 "신청" 버그 — ✅ 해결됨 (2026-07-08)

> **상태: 해결.** 원인 = **세션 프라이밍 누락**. `saveRequestData` POST 전에 같은 세션으로
> `GET /view/pss/requestData` 를 해야 신청이 '신청내역에 등록'된다. 안 하면 200·응답 1/2 가
> 와도 **조용히 미등록**(=승인/다운로드 불가). → `sgis_request.prime_session()` 신설,
> `submit_cart()`·CLI 제출 직전 자동 호출로 수정(커밋 예정).
> ⚠️ 이 파일엔 쿠키·계정정보(아이디/이메일/전화) 절대 기록하지 않는다.

## 0. 해결 요약 (2026-07-08, 국내 IP 실측)
- **원인 오해였던 것**: "응답 `2`=실패"는 오진. **응답 `2`여도 프라이밍만 하면 정상 등록**된다.
- **진짜 원인**: 우리 코드는 `requestData` 페이지를 GET 하지 않고 `saveRequestData` 를 바로 POST.
  서버는 세션이 신청화면을 거쳤을 때만 등록 처리 → 프라이밍 없으면 미등록(응답은 1/2 아무거나).
- **실측 확정**: 프라이밍 전 25011/25012 = 미등록. GET requestData 후 동일 25011 재시도 = **등록(80421)**.
  2쿠키(JSESSIONID+accessToken)만으로 프라이밍·등록 성공. `submit_cart` 경로로 청주 등록 확인(80423).
- **후속**: 어제(07-07) '안 올라갔다'던 전국 258건은 사실 **다 등록돼 있었음**(신청내역에 존재).
  당시 미등록으로 보인 배치들은 프라이밍 누락 케이스였을 것.

## 1. 증상
- 앱/스크립트로 SGIS 자료제공 **신청**을 보내면 서버 응답이 항상 **`2`** 로 온다.
- `2` = "저장되었습니다"(임시저장). **`1` = "신청되었습니다"(진짜 접수)** 와 다르다.
- 응답 `2` 는 **신청내역(requstDataList)에 행이 생기지 않는다** → 승인 대상이 안 됨 → 다운로드 불가.
- 즉 **응답 2 는 사실상 실패**(우리 목적 기준). 겉으론 200 OK라 성공처럼 보였을 뿐.

## 2. 지금까지 확정된 사실 (2026-07-07, 사용자 PC·국내 IP에서 실측)
| 테스트 | 결과 |
|---|---|
| 한 신청에 5·12·20·30·40·50·89개 항목 | **전부 2** |
| 한 번도 안 건드린 새 시군구(합천·괴산·강릉 등) 1항목만 | **2, 내역 39→39 변화 없음** |
| 폼(censusFm)의 **모든 필드**를 브라우저와 동일하게 채워 전송 | **2** |
| `param_userkey`(로그인 계정 아이디)까지 채워 전송 | **2** |
| `sgis_census_level_area=all`, `..._check=Y` 추가 | **2** |
| **브라우저에서 사람이 직접 시군구 1개 신청** | **성공(신청됨)** ✅ |

### 결론
1. **응답 2 = 진짜 신청 아님** (예전 커밋 `cae20cf`의 "2도 성공" 판정은 **오판 → 정정 필요**).
2. **개수 문제 아님** — 1개도 2. 잘게 쪼개 250번 반복해도 250번 다 2가 나옴.
3. **하루 신청 한도(캡) 문제 아님** — 브라우저 수동 1건은 지금도 성공하므로.
4. → **남은 원인: 우리가 보내는 `saveRequestData` 요청이 브라우저 요청과 미세하게 다르다.**
   필드는 다 맞췄으므로 멀티파트 인코딩 / 헤더 / 토큰 / 순서 등 미세 차이로 추정.

## 3. 다음 단계 (최종 열쇠)
브라우저에서 **성공하는** 신청의 실제 요청을 캡처해 우리 것과 1:1 비교한다.
1. 크롬 **F12 → Network** 탭
2. 필터칸에 `saveRequestData` 입력
3. **시군구 1개 손으로 신청**(성공하는 그 동작)
4. 뜬 `saveRequestData` 줄 우클릭 → **Copy → Copy as cURL**
5. 그 요청 본문/헤더를 `sgis_request.py`의 `build_body()` / `submit()` 과 대조 →
   빠졌거나 다른 필드·헤더를 찾아 반영 → 응답이 `1` 로 바뀌는지 확인.

## 4. 고칠 코드 위치
- `sgis_request.py`
  - `build_body(cart, boundary, applicant)` — 멀티파트 본문 생성부(여기 필드 추가/수정 유력)
  - `submit(cookie, body, boundary)` — 헤더 지정부
- `app_v2.py`
  - `_apply_render_results(results)` — 현재 `1`,`2` 둘 다 성공 처리 중.
    **원인 수정 후엔 `1`만 성공으로 되돌려야 함**(2는 실패로 경고).

## 5. 버튼 구조 정리 (중복 아님, 겹침 있음)
- **좌측 "전국 디폴트 원스톱"** = 신청 + (승인 후)자동 다운로드 + 엑셀 250여 개 빌드까지 한 화면에서 끝까지. (상위 버전)
- **"선택 시군구 신청" 옆 "전국 전체 시군구 신청"** = 신청만 250여 개 자동 투입.
- 둘 다 지금 응답 2가 뜨는 건 **같은 신청 버그가 원인**. 그 버그 하나 고치면 **두 버튼 다 정상**.

## 6. 이미 완성돼 작동하는 부분 (문제 아님)
- 시군구별 zip 다운로드 → 시군구별 엑셀 빌드 **250여 회 반복 루프**:
  `batch_build.py`(빌드) + `batch_watch.py`(승인 감시→다운→빌드 무인) + `sgis_request.fetch_download_list/download_zip`(다운로드).
- **신청(응답 1)만 되면** 이 뒷단은 그대로 굴러감.
